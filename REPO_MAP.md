# Repository Map

## Purpose
Provide a quick orientation for humans and AI agents navigating the repository.

## Top-Level Structure
- `README.md` — High-level description and navigation.
- `SYSTEMS.md` — Central index of automation systems.
- `systems/` — Per-system overviews, interfaces, designs, evaluations, prompts.
- `docs/` — Architecture standards (`system-philosophy`, `system-interface-spec`, `system-lifecycle`, `system-map`, `system-status-registry`, `system-failure-modes`).
- `schemas/` — Authoritative schema definitions; inventory in `schemas/README.md`.
- `prompts/` — Prompt registry aligned to systems; index in `prompts/README.md`.
- `eval/` — Evaluation harness scaffolds; matrix in `eval/test-matrix.md`.
- `workflows/` — Conceptual workflow descriptions.
- `issues/` — Research questions and backlog items.
- `examples/` — Illustrative artifacts.
