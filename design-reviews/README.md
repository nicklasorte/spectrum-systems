# Design Reviews

Purpose: store architecture and design review artifacts for this repository in a consistent, automatable structure.

Usage:
- Create each review as its own markdown file in this directory.
- Name every review file using `YYYY-MM-DD-<short-scope>.md` to keep ordering deterministic (example: `2026-03-14-architecture-review.md`).
- Use `design-reviews/review-template.md` and the canonical sections defined in `docs/design-review-standard.md`.
- After publishing a review, link it in `docs/review-registry.md` and extract actions per `docs/review-to-action-standard.md`.
