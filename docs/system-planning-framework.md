# System Planning Framework

## Overview
Every automation system designed in this repository must follow the planning steps below before implementation begins. All steps must be completed and documented prior to any build work.

## Planning Steps
1. Identify high-value bottlenecks
2. Distinguish root bottlenecks from symptoms
3. Define system boundaries
4. Select a wedge problem
5. Define input schemas
6. Define the artifact chain
7. Determine where AI should be used
8. Define human review checkpoints
9. Build evaluation harnesses
10. Create institutional memory storage
11. Ensure outputs are report-ready
12. Use modular system architecture
13. Document the house methodology
14. Create a ranked system roadmap
15. Protect dedicated system design time

These planning steps must be completed before implementation starts to ensure deterministic, testable system designs.
