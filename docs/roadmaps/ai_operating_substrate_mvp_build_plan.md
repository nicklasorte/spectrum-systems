# AI Operating Substrate MVP Build Plan

## Scope
This plan defines only the first narrow 10-step dependency-valid build wave for MVP substrate + minimum artifact intelligence.

It does **not** implement the full taxonomy, broad autonomy, or broad model/workflow expansion.

| ID | Prompt | Status | What It Does | Why It Matters | Which artifact(s) it introduces | Trust / measurement / governance gain unlocked | Dependencies / hard gate |
| --- | --- | --- | --- | --- | --- | --- | --- |
| AOS-MVP-01 | PLAN — Task lifecycle governance contract slice | Planned | Define and approve contract-first scope for task-level lifecycle governance and registry linkage. | Prevents ad hoc prompt-to-task execution paths. | plan artifact only | Governance scope lock and non-drift execution boundary. | Must complete before AOS-MVP-02..03. |
| AOS-MVP-02 | BUILD — Add `task_registry_entry` contract + example + manifest pin | Planned | Create canonical task registry artifact with lifecycle states and prompt-family linkage. | Enables governed task lineage and admission/promotion semantics. | `task_registry_entry` | Trust: lifecycle governance; Measurement: task lineage coverage. | AOS-MVP-01 approved. |
| AOS-MVP-03 | WIRE — Enforce prompt→task linkage in model request assembly | Planned | Require governed paths to resolve prompt versions through task registry identity before request emission. | Blocks prompt-only bypass; enforces task-level governance. | request lineage extension in `ai_model_request` usage | Governance: non-bypassable lifecycle enforcement. | AOS-MVP-02 complete. |

## Hard-gate policy for this plan
- Steps that broaden model breadth, autonomy, or artifact-family breadth are blocked until AOS-MVP-10 passes.
- Any bypass of task registry, routing/context artifacts, or eval-entry coverage is fail-closed.
