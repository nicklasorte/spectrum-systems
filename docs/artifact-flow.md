# Artifact Flow

The ecosystem moves policy work through governed, structured artifacts—not ad hoc documents—so every handoff stays deterministic and contract-aligned. Contract IDs and versions are pinned in `contracts/standards-manifest.json`, and systems are mapped in `SYSTEMS.md` and `docs/ecosystem-map.md`.

## Primary Artifact Pipeline

Working paper → reviewer comments → comment resolution matrix → adjudicated matrix → annotated DOCX → meeting transcript → meeting minutes → program advisory artifacts.

### Lifecycle Diagram

```mermaid
flowchart LR
    WP[Working paper] --> RC[Reviewer comments]
    RC --> CRM[Comment resolution matrix]
    CRM --> AM[Adjudicated matrix]
    AM --> ADOCX[Annotated DOCX]
    ADOCX --> MTX[Meeting transcript]
    MTX --> MM[Meeting minutes]
    MM --> PAA[Program advisory artifacts]
```

## Artifact Responsibilities and Contracts

All artifacts live in approved external storage with an accompanying `external_artifact_manifest` entry (see `docs/external-storage-implementation-guide.md` and `docs/data-boundary-governance.md`). Systems consume and emit contracts declared in `contracts/standards-manifest.json`; storage paths and checksums stay outside GitHub.

| Artifact | Producing system | Consuming system | Governing contract/schema | Storage policy |
| --- | --- | --- | --- | --- |
| Working paper | Upstream authoring; ingested by `working-paper-review-engine` (SYS-007) | `working-paper-review-engine` (SYS-007) | `working_paper_input` (see `contracts/examples/working_paper_input.json`) | Store source PDF/DOCX in external storage; register in `external_artifact_manifest` with checksum and versioned paper ID. |
| Reviewer comments | Normalized by `working-paper-review-engine` (SYS-007) | `comment-resolution-engine` (SYS-001) | `reviewer_comment_set` (standards manifest) | Keep normalized JSON + provenance sidecar in external storage; manifest entry must link back to source working paper and reviewer IDs. |
| Comment resolution matrix (initial) | Seeded by `working-paper-review-engine` (SYS-007) | `comment-resolution-engine` (SYS-001); `docx-comment-injection-engine` (SYS-008); `spectrum-pipeline-engine` (SYS-009) | `comment_resolution_matrix_spreadsheet_contract` (`contracts/schemas/comment_resolution_matrix_spreadsheet_contract.schema.json`) | Persist spreadsheet in external storage with manifest record and checksum; track linkage to originating reviewer_comment_set. |
| Adjudicated matrix | Produced by `comment-resolution-engine` (SYS-001) | `docx-comment-injection-engine` (SYS-008); `spectrum-pipeline-engine` (SYS-009) | `comment_resolution_matrix_spreadsheet_contract` + `comment_resolution_matrix` (standards manifest) | Store adjudicated matrix and run manifest externally; manifest must include disposition metadata, adjudicator identity, and upstream matrix hash. |
| Annotated DOCX | Produced by `docx-comment-injection-engine` (SYS-008) | `spectrum-pipeline-engine` (SYS-009); review participants | `pdf_anchored_docx_comment_injection_contract` (`contracts/schemas/pdf_anchored_docx_comment_injection_contract.schema.json`) | Save annotated DOCX and audit manifest to external storage; manifest must bind anchors to adjudicated matrix row IDs. |
| Meeting transcript | Captured by meeting recording/transcription pipeline; normalized for `meeting-minutes-engine` (SYS-006) | `meeting-minutes-engine` (SYS-006) | Transcript input expectations in `systems/meeting-minutes-engine/interface.md` (timestamped speaker turns) | Store raw audio/text externally with manifest entry; include transcript version, timestamp source, and checksum. |
| Meeting minutes | Generated by `meeting-minutes-engine` (SYS-006) | `spectrum-pipeline-engine` (SYS-009); `spectrum-program-advisor` (SYS-005) | `meeting_minutes_contract` (`contracts/meeting_minutes_contract.yaml`) | Keep JSON, DOCX, and validation report in external storage; manifest ties minutes to transcript and agenda seed hashes. |
| Program advisory artifacts | Generated by `spectrum-program-advisor` (SYS-005) | Governance boards and program managers | Readiness contracts (`program_brief`, `study_readiness_assessment`, `next_best_action_memo`, `decision_log`, `risk_register`, `assumption_register`, `milestone_plan` from standards manifest) | Store advisory bundle externally with manifest capturing all inputs (minutes, matrices, agendas) and readiness artifact checksums. |
