RDX-003 — BATCH-M: Control Authorization for Roadmap-Selected Batch

- Add storage/validation helpers for the authorization artifact.
- Add focused tests for allow/warn/freeze/block/determinism/non-execution boundary.
- Update process-flow documentation generation to include Control Authorization between Roadmap Selection and Execution.

## Invariants to preserve
- Artifact-first architecture with schema-bound outputs.
- Deterministic behavior (same inputs => identical outputs).
- Fail-closed behavior on missing/ambiguous/invalid inputs.
- No hidden decision logic; use explicit structured reason codes.
- No duplicate control path; selection proposes, control authorizes, execution remains downstream.
- No PQX execution, roadmap mutation, or scheduler/autonomous loop additions.

## Contracts touched
- `roadmap_execution_authorization` (new, `1.0.0`).
- `standards_manifest` metadata version bump and contract registration update.

## Risks
- Reason-code vocabulary drift between code and schema.
- Manifest registration mismatch causing contract-enforcement failures.
- Process-flow updates drifting from generated style expectations.

## Acceptance criteria
- New authorization artifact validates against schema and includes required fields.
- Authorizer deterministically returns `allow|warn|freeze|block` with `authorized_to_run` and structured reasons.
- Missing/invalid prerequisites fail closed (`freeze`/`block`).
- Tests prove no execution side effects and deterministic replayability.
- Required validation commands pass.
